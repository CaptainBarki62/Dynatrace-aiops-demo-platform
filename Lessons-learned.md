# Lessons Learned

1. EKS can be ACTIVE while worker nodes are unavailable.

2. Dynatrace CRD version must match Operator version.

3. Webhooks must be healthy before Dynakube deployment.

4. OneAgent supports only one deployment per node.

5. Helm deployment is easier than raw manifests.

6. AWS SSO authentication must be configured before CLI access.

7. Synthetic failures create reliable demo scenarios.

8. Workflow automation provides immediate business value.

9. OpenTelemetry can complement or replace portions of OneAgent monitoring.

10. Self-healing demos resonate strongly with customers.
